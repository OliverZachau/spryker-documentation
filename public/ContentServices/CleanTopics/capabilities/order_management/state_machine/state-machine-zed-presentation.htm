To add manual trigger button or history for items, you need to add some implementation. State machine provides methods for triggering events and for retrieving history. StateMachineFacade::processedStateMachine(StateMachineItemTransfer[])updates the given array of items with data from state machine persistence. This method is useful when you are not storing all the data for the StateMachineItemTransfer into the local table. GivenStateMachineItemTransfer[]with (idItemState, identifier, idStateMachineProcess) it will fill other data. StateMachineFacade::getManualEventsForStateMachineItems(StateMachineItemTransfer[])will return the array containing the mapping of the manual events, where the key is the local entity identifier. This can be used for adding a manual triggering button. StateMachineFacade::getStateHistoryByStateItemIdentifier($idStateMachineProcess, $identifier)will return history of a given item, array ofStateMachineItemTransfer[]. This can be used to display past event transitions that happened for the item.
