Database Schema The Product Review module adds a new table:spy_product_review. The table stores all product review related information such as description, rating, locale, customer, product, etc. Product Reviews in Search product-review typeis defined under the already existingindexin Search. The product review Search documents are generated with the newly createdproduct-review typeusing the configurable collectors with the following mapping: { "mappings": { "product-review": { "dynamic_templates": [ { "search-result-data": { "mapping": { "index": "no" }, "path_match": "search-result-data.*" } } ], "properties": { "search-result-data": { "type": "object", "include_in_all": false, "properties": { } }, "store": { "type": "string", "include_in_all": false, "index": "not_analyzed" }, "locale": { "type": "string", "include_in_all": false, "index": "not_analyzed" }, "id-product-abstract": { "type": "integer", "include_in_all": false }, "rating": { "type": "integer", "include_in_all": false }, "created-at": { "type": "string", "include_in_all": false, "index": "not_analyzed" } } } } } TheProductReviewsQueryPluginallows to alter a Search query to look for the product reviews of a specific product abstract with the separateproduct-review type. Configurable Collectors Collectors are able to manage documents within different indexes and to use different types usingAbstractConfigurableSearchPropelCollectorandAbstractConfigurableSearchPdoCollectorabstract classes.
